# Contributing / תרומה לפרויקט

Thanks for helping improve BavliManager.

תודה על הרצון לתרום ל-BavliManager.

## Development setup
- Use JDK 17 and Maven 3.8+
- Run `mvn clean test verify` before opening a PR

## Workflow
1. Create a branch from `main`
2. Keep commits focused and descriptive
3. Open a pull request with clear summary and testing notes

## Code style
- Keep changes minimal and targeted
- Preserve existing naming and structure unless refactor is required
- Add tests for new behavior when feasible

## Pull request checklist
- [ ] Code builds successfully
- [ ] `mvn test` passes
- [ ] `mvn verify` passes
- [ ] Documentation updated (if needed)

## Setup מקומי
- להשתמש ב-JDK 17 ו-Maven 3.8+
- להריץ `mvn clean test verify` לפני פתיחת PR

## תהליך עבודה
1. יצירת branch מ-`main`
2. קומיטים ממוקדים עם הודעות ברורות
3. פתיחת PR עם תיאור שינוי ובדיקות שבוצעו
